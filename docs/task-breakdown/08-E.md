### T-19 — Groundwork: security/referral design + event schema (prep R3)

**Description:** งานเตรียม R3 ระหว่าง R2: ออกแบบ event schema กลางสำหรับเหตุการณ์ความปลอดภัย (เข้า-ออก, เหตุผิดปกติ) และ flow การส่งต่อ (referral) ร่วมกับ Module F — กำหนดประเภท event, ผู้มีสิทธิ์บันทึก/เห็น ตาม role-permission matrix ไม่ใช่ feature ที่ user ใช้

**Definition of Done:**
- Event schema (ประเภท, payload, ระดับความลับ) ผ่าน review Lead + สอดคล้อง T-02
- Referral flow design (สาเหตุ: ศูนย์เต็ม/ทรัพยากรขาด/medical, สถานะ, ผู้รับผิดชอบ) ตกลงร่วมกับ Module F แล้ว
- ข้อสรุปเพียงพอให้ T-33/T-34 เริ่ม build โดยไม่ redesign

### T-33 — Security check-in/out + safety monitoring (people+pets) (FR-47)

**Description:** ระบบความปลอดภัยของศูนย์ตาม source Module E: ติดตามการเข้า-ออกศูนย์ระดับบุคคล (ต่อยอด check-in/out ครัวเรือน T-06) ทั้งคนและสัตว์เลี้ยง + monitoring ภาพรวม "ใครอยู่ในศูนย์ตอนนี้" และบันทึกเหตุการณ์ผิดปกติ

**Definition of Done:**
- Security Event แยกจาก movement event ปกติ (FR-12/13 baseline + T-06) แต่ reference ถึงกันได้ (PRD FR-47); ครอบทั้งคนและสัตว์ (ผูก Pet Record T-07)
- หน้า monitoring แสดงจำนวนคน/สัตว์ที่อยู่ในศูนย์ปัจจุบัน + รายการออกแล้วยังไม่กลับเกินกำหนด
- บันทึกเหตุการณ์ผิดปกติ (incident) พร้อมระดับและผู้รับผิดชอบ จำกัดการเห็นตาม role
- Test ยอดเข้า-ออกสอดคล้อง occupancy + demo flow เข้า-ออก-เกิดเหตุ 1 รอบ
