### T-40 — Search consent / opt-out (FR-52, FD-12)

**Description:** เก็บ consent ของผู้พักพิงว่ายินยอมให้ปรากฏในผลค้นหาสาธารณะหรือไม่ ตั้งแต่ขั้นลงทะเบียน (default ตาม FD-12) และเปลี่ยนใจ opt-out ได้ทุกเมื่อ — กลุ่มเสี่ยง (เช่น ผู้หนีภัยความรุนแรงในครอบครัว) ต้องถูกซ่อนจากการค้นหาได้จริง

**Definition of Done:**
- บันทึก consent ต่อบุคคลตอนลงทะเบียน + แก้ภายหลังได้ (โดยเจ้าตัวผ่านเจ้าหน้าที่) พร้อมประวัติการเปลี่ยน
- Opt-out มีผลทันทีกับ T-41 — record หายจากผลค้นหา (test ยืนยัน ไม่มี cache ค้าง)
- ค่า default + ถ้อยคำขอ consent ตรงตาม FD-12 / ที่ DPIA (P-03) กำหนด
- Audit การเปลี่ยน consent ครบ + demo opt-out แล้วค้นไม่เจอ

### T-41 — Privacy-preserving public family search + anti-enumeration (FR-53)

**Description:** หน้าเว็บสาธารณะให้ญาติค้นหาผู้ประสบภัย (ไม่ต้อง login) — ผลลัพธ์เปิดเผยขั้นต่ำ: ยืนยันว่า "พบ และอยู่ศูนย์ใด" โดย mask ข้อมูลส่วนตัวอื่น พร้อมกลไก anti-enumeration กันการกวาดข้อมูล (rate-limit, ต้องพิมพ์ชื่อใกล้เคียงจริง, ไม่มี list-all) ตาม design P-03

**Definition of Done:**
- ค้นจากชื่อ-นามสกุล (fuzzy ตามเกณฑ์ P-03) → แสดงข้อมูลขั้นต่ำตาม PRD FR-53: "พบ/ไม่พบ" + ศูนย์ (ระดับที่ปลอดภัย) + ช่องทางติดต่อเจ้าหน้าที่ — **ไม่เปิด** medical, national ID, ที่อยู่/เบอร์ติดต่อ; ระดับการเปิดชื่อ masked ให้เป็นไปตาม P-03/DPIA
- เฉพาะคนที่ consent (T-40) เท่านั้นที่ปรากฏ; ไม่พบ กับ ไม่ยินยอม ต้องแยกไม่ออกจากภายนอก
- Anti-enumeration: rate-limit ต่อ IP/session, CAPTCHA, query สั้นเกิน/กว้างเกินถูกปฏิเสธ, ไม่มี endpoint ไล่ดึงรายชื่อ (test เชิง adversarial ครบ)
- ทุก public query ลง audit (ไม่เก็บ PII ผู้ค้น แต่ trace พอสำหรับ abuse detection — FR-53), mobile-first, ภาษาไทยเป็นหลัก
- ผ่าน data-governance review + DPIA ก่อนเปิด public (feature-specific NFR ใน R4 PRD)
- Demo ค้นเจอ ↔ opt-out แล้วค้นไม่เจอ end-to-end
