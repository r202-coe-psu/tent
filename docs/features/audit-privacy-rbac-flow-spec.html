Audit Surfaces

  • Registration create/update, duplicate override, fast-track override, medical update, movement event, export request, offline review decision, and permission-relevant admin changes must all produce audit evidence
  • Audit metadata should capture actor, action, entity type/id, shelter scope, request correlation, and non-sensitive summary; it must not dump raw medical or raw identifier payloads
  • QA and release review need audit as evidence, not just a background table no one can verify

Acceptance Criteria

  • Unauthorized role cannot retrieve raw medical fields or `national_id_hash` via direct API call
  • Export action always stores requester, scope and masking policy snapshot
  • Duplicate override, fast-track override and closed-shelter override each require reason and create audit evidence
  • Search-excluded record stays out of normal search but remains operable via approved privileged workflow

Open Questions

  • Final Role Permission Matrix by role/action/field is still pending proposal and approval
  • Whether audit review UI is part of July user-facing surface or only release/admin tooling still needs explicit decision
  • Exact export field masks by role must be fixed in API contract or policy artifact
