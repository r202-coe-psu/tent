Feature Flow Spec

Public Portal — Shelter Dashboard (ตรวจสอบศูนย์พักพิง)

Back Landing Spec Draft v0.1 2026-06-22

Summary

หน้า "ตรวจสอบสถานะศูนย์พักพิง" (PUBLIC SHELTER DASHBOARD) — แยกออกมาจาก Public Portal landing spec (nav link 1). เป็น จุดที่ประชาชน / ญาติผู้ประสบภัยใช้ดู รวมศูนย์พักพิง สถานะความจุ และเปอร์เซ็นต์ความหนาแน่นของผู้ประสบภัยตามเวลาจริง เพื่อ ประกอบการตัดสินใจเคลื่อนย้ายและขอรับความช่วยเหลือ. Layout = metric cards (บนสุด) + 3 คอลัมน์: panel ค้นหา/ฟิลเตอร์ (ซ้าย), map พร้อมหมุดศูนย์ (กลาง), list ศูนย์พักพิง (ขวา).

หน้านี้เป็น Public / No-Login Tier (PUB) — ต้องไม่มีข้อมูลส่วนบุคคลใด ๆ ปรากฏ. ข้อมูลที่แสดงเป็น aggregate ต่อศูนย์เท่านั้น (occupancy รวม ไม่มีรายบุคคล). รายละเอียด trust boundary ดูที่ Public Tier Flow Spec
ออกแบบให้ใช้งานได้บนมือถือในสภาวะ network ไม่เสถียร — list/metrics ต้องอ่านได้แม้ map ยังโหลดไม่เสร็จ; รองรับ TH/EN (EN toggle)
Change tracking: หน้านี้แยกออกมาตาม CR-005 §D. metric "ผู้พักพิงปัจจุบัน" / "กลุ่มเปราะบาง" approved ให้แสดงได้ (OP-6 / OP-8); flag เหลือเป็น kill-switch — ดูส่วน Data Exposure & Privacy

Metrics Behavior

  • Data source: GET /public/v1/transparency/summary — aggregate counts (central-only)
  • Refresh: polling ทุก 10 นาที (OP-7), stale-data threshold 30 นาที — สอดคล้องกับ landing metrics
  • Loading: skeleton ต่อ card; ไม่บล็อก paint ของ list/filter
  • Fail/offline: แสดง "—" แทนตัวเลข + tooltip "ข้อมูลสดไม่พร้อมใช้งาน"

3 · Search & Filter Panel (ซ้าย)

Panel "ค้นหา" — desktop คอลัมน์ซ้าย; mobile = collapsible ด้านบน list. ทุก filter เป็น client-side query param เท่านั้น — ไม่ใช่ data exposure.

Filter Behavior

  • filter ทำงานกับ dataset เดียวกันที่ขับ list + map — เปลี่ยน filter อัปเดตทั้งสองพร้อมกัน
  • "ระยะทางเส้นทาง" ต้องมีตำแหน่งผู้ใช้ก่อน; ถ้า deny geolocation → ซ่อน/disable distance filter + แสดงทั้งหมด
  • จำนวนผลลัพธ์แสดงที่หัว list ("รายชื่อศูนย์พักพิง N แห่ง")
  • empty state: "ไม่พบศูนย์ตามเงื่อนไข" + ปุ่มล้างฟิลเตอร์

4 · Map (กลาง)

  • แสดงหมุดศูนย์พักพิงตาม geo ของแต่ละศูนย์ (lat/lng) + หมุด "เป้าหมาย" และเส้นทางตัวอย่าง
  • พิกัด/ที่ตั้งศูนย์เป็น ข้อมูลสาธารณะเชิงปฏิบัติการ — expose ได้ (ไม่ใช่ PII)
  • คลิกหมุด → highlight card ใน list (และ scroll ไปหา); คลิก card ใน list → pan/zoom ไปหมุดนั้น (sync 2 ทาง)
  • marker สีตามสถานะความหนาแน่น (ดู occupancy_pct ใน list) — แดง ≥90%, ส้ม 70–89%, เขียว <70%
  • map degrade gracefully: ถ้า tile โหลดไม่ได้ (offline) → list + metrics ยังใช้ได้เต็มที่; แสดง placeholder แทน map
  • ปุ่ม "นำทาง" ใน list เปิด external map app (geo:/Google Maps directions) — ออกนอกระบบ

List Behavior

  • progress bar สีตาม occupancy_pct: แดง ≥90%, ส้ม 70–89%, เขียว <70%
  • per-shelter occupancy/capacity แสดงตัวเลขเป๊ะเสมอ (OP-9 approved — เช่น 3/250, ไม่ปัด/ไม่ mask แม้ count ต่ำ)
  • เรียงลำดับ default: ระยะทางใกล้สุดก่อน (ถ้ามีตำแหน่ง) มิฉะนั้นตามชื่อ
  • การ์ด sync กับ map marker (hover/click highlight)

Accessibility & Performance Requirements

  • WCAG 2.1 AA — contrast ≥ 4.5:1; metric บน dark hero และ progress bar labels ต้องอ่านได้
  • Search input + filter selects ต้องมี aria-label / associated label
  • map ต้องมี non-map fallback (list) — ผู้ใช้ screen reader เข้าถึงข้อมูลศูนย์ผ่าน list ได้ครบ
  • metric ที่อัปเดตเอง ใช้ aria-live="polite"
  • shelter card actions ใช้ <a> / <button> ที่ focusable
  • LCP ≤ 2.5 s บน 3G; map (tile โหลดหนัก) ต้อง lazy / ไม่บล็อก paint ของ list/metrics
  • ภาษาไทย: lang="th" บน root เมื่อ locale = TH

UAT Checklist

  • ☐ เปิด /shelters โดยไม่ login → เห็น hero + metric cards + filter + map + list
  • ☐ metric cards แสดงตัวเลข aggregate จาก API; card ที่ flag off ไม่ปรากฏ
  • ☐ เลือกจังหวัด/อำเภอ/ตำบล → list + map อัปเดตพร้อมกัน
  • ☐ เลือกระยะทาง 5/10/20/50 (เมื่อมีตำแหน่ง) → กรองรัศมีถูกต้อง
  • ☐ ติ๊ก/เอาออก "เปิดใช้งาน"/"ปิดใช้งาน" → list กรองตามสถานะ
  • ☐ คลิกหมุด map → highlight card ที่ตรงกันใน list (และกลับกัน)
  • ☐ per-shelter แสดง occupancy/capacity (เช่น 3/250) + จำนวนที่ว่าง ถูกต้อง
  • ☐ คลิก "ดูรายละเอียด" → ไป /shelters/{id}
  • ☐ คลิก "นำทาง" → เปิด external map directions
  • ☐ filter ไม่พบผล → empty state + ปุ่มล้างฟิลเตอร์
  • ☐ ปิด network → metrics "—", map placeholder, list empty + toaster; filter UI ยังกดได้
  • ☐ deny geolocation → distance filter + ระยะทางซ่อน/disable
  • ☐ Toggle EN → ทุก label/หน่วยเปลี่ยนเป็น English
  • ☐ Mobile 375px → filter collapsible, map/list stack อ่านได้
  • ☐ card "ผู้พักพิงปัจจุบัน" + "กลุ่มเปราะบาง" แสดงค่าจริง (OP-6/OP-8 approved); ทดสอบ kill-switch: ตั้ง flag off → card นั้น ไม่ปรากฏ และ API ไม่คืนค่า
