Screen Flow

  • `Login -> Shelter Context Gate -> Home Shell` คือ happy path ของทุก role
  • ถ้า user มี assigned shelter เดียว ให้ระบบ set active shelter อัตโนมัติและ redirect ทันที
  • ถ้า user ไม่มี assigned shelter และไม่ใช่ system admin ให้ deny ทั้ง UI และ API
  • ทุก protected page ต้องเช็ค expired session ก่อน render feature data
  • หน้า Shelter Master มีเฉพาะ admin/system role ที่ได้รับสิทธิ์; registration/medical/viewer เห็นได้แค่ shelter metadata ที่จำเป็นต่อ workflow

API and Data Intent

  • Auth endpoints: `POST /auth/login`, `POST /auth/logout`, `GET /auth/me`
  • Shelter context must return actor identity, `role_keys`, `assigned_shelter_ids`, and current shelter metadata needed by the shell
  • Shelter master endpoints must own `shelters.status`, `capacity.total`, `capacity.soft_limit`, `allow_over_capacity`
  • 403 path must return stable permission failure shape so UI can render blocked state instead of generic error
  • Audit must capture login success/failure where policy requires, shelter status changes, and override reasons

Acceptance Criteria

  • User with wrong credentials stays on Login and sees explicit failure state
  • User with one assigned shelter lands in the correct scope without manual chooser
  • User cannot read or mutate data outside assigned shelter by direct API call
  • Closed shelter blocks normal new check-in unless authorized override path is used with reason
  • Session expiry returns user to Login without exposing stale protected content
